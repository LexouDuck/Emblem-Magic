Here is how the header of an MMF file is laid out:
FE7U # FE Game identifier (FE7U, FE6J, etc)
something Editor # Title of the module (shown as window title)
by whomever # Author of the module (shown on bottom bar)
0xFFFFFF # Address of the array (will prompt the user for an address if 0x000000)
100 # Amount of entries in the array
16 # Length of each entry (in bytes)
LIST # The type of control for the array, usually LIST or POIN
file.txt # The TXT file for the array entry names
 Each subsequent field for the module is like so:
[text] # Name of the field
[text] # Category of the field (if the Category is 'Unused', it will be the last category to show up)
[number] # byte/bit index of the field (add a 'b' at the end of a number to make it a bit amount)
[number] # byte/bit length of the field
[NULL] # The kind of control, see more about these just below
[file.txt] # usually 'NULL', or the filename of a txt list file to use for the control
 Here is the list of possible controls for fields: NULL # nothing
BOOL # CheckBox
TEXT # TextBox (one-line)
HEXT # HexBox (multi-line)
HEXU # Hex Numeric Up/Down box
NUMU # Unsigned decimal NUDbox
NUMS # Signed decimal NUDbox
LIST # ByteBox with dropdownlist, should have a TXT list file
POIN # PointerBox (with a dropdown list if the TXT file is not 'NULL')
